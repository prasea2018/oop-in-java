## **2. Key Concepts (with clear explanations)**

### **Switch-case**

A control statement used to execute different parts of code based on the **value of an expression**.
It acts like a **multi-way branch**.

### **Case Labels**

Each `case` specifies a constant value. If the switch expression matches the case label, that block executes.

### **Break Statement**

Stops the execution of further cases. Without it, Java continues into the next cases (fall-through).

### **Default Case**

Executes if **no case matches**. Works like the "else" in if-else.

### **Fall-through**

If a case does NOT end with `break`, Java continues executing the next cases sequentially.

### **Allowed Case Types**

Java allows:

- `byte`
- `short`
- `int`
- `char`
- `enum`
- `String` (Java 7+)

Floating-point types (`float`, `double`) are **NOT allowed**.

### **Efficiency Difference: switch vs. if-else**

- **If-else** checks conditions **one by one** â†’ slower for many conditions
- **Switch-case** jumps **directly** to the matching value â†’ faster

### **Menu-driven Programming**

Switch-case is excellent for software that shows menu options (File > Save, Edit > Copy).

## **4. Detailed Breakdown**

### **A. What is Switch Case?**

A switch-case is a branching statement that chooses a block of code to run based on the **value** of an expression (usually a variable like day-number, menu-option, etc.).

### **B. Syntax**

```java
switch (expression) {
    case value1:
        // statements
        break;

    case value2:
        // statements
        break;

    default:
        // statements
}
```

### **C. How It Works**

If `expression` evaluates to a value, Java:

1. **Jumps directly** to the matching case
2. Executes that block
3. Stops at `break`

If no case matches â†’ goes to **default**

### **D. Example Explained**

```java
int day = 1;

switch(day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    default:
        System.out.println("Invalid day");
}
```

If `day = 1` â†’ prints â€œMondayâ€

If `day = 10` â†’ no matching case â†’ prints â€œInvalid dayâ€

### **E. What If Break Is Missing? (Fall-through)**

```java
case 1:
    System.out.println("Monday");
case 2:
    System.out.println("Tuesday");
    break;
```

If day = 1
â†’ prints Monday
â†’ FALLS THROUGH
â†’ prints Tuesday
â†’ stops at break

This technique is rarely needed but useful when multiple cases require the **same logic**.

### **F. Order of Cases**

You can write cases in **any order**; order doesnâ€™t matter.

### **G. Allowed Case Values**

Valid:

- `case 1:` (int)
- `case 'A':` (char)
- `case "Open":` (String)
- `case BYTE_VALUE:`

Invalid:

- `case 1.5:` (float not allowed)
- `case x + 1:` (must be a constant, known at compile time)

### **H. Switch vs If-Else**

Switch is better when:

- Many exact matches (1â€“7, â€œOpenâ€, â€œSaveâ€â€¦)
- Menu systems
- Better readability
- Faster execution because it directly jumps to the case

If-else is better when:

- You need **ranges**
  Example: `if (x > 10 && x < 20)`

Switch cannot handle ranges.

### **I. Menu-driven Programs**

Most applications have menus:


```java

```

Using `String` cases (Java 7+), programming menus becomes easy and readable.

## **5. Summary**

- **Switch-case** is a multi-way branching statement.
- It evaluates an expression and jumps **directly** to the matching `case`.
- Each case typically ends with **break** to prevent fall-through.
- **Default** handles unmatched values.
- Allowed types include: `int`, `char`, `String`, `byte`, `short`, `enum`.
- Switch-case is faster and cleaner than many `if-else` conditions.
- Ideal for **menu-driven applications** and exact-value selections.
