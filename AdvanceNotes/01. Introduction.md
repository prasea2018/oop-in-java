## **2. Key Concepts (with Clear Explanations)**

### **JDK (Java Development Kit)**

* A toolkit for **developing Java applications**.
* Contains:

  * **javac** Java compiler
  * **java**  Java launcher (runs programs)
  * **jdb**   debugger
  * Library files, documentation tools, etc.

### **JRE (Java Runtime Environment)**

* Provides everything needed to **run** a Java program.
* Contains:

  * Java class libraries
  * JVM
  * Runtime utilities
* Installed automatically when you install JDK.

### **JVM (Java Virtual Machine)**

* The engine that **executes Java bytecode**.
* Platform-dependent implementation but executes platform-independent bytecode.
* Part of the JRE.

### **How Java Works (Process)**

1. You write a file ? `First.java`
2. **javac First.java** ? compiler converts code ? `First.class`
3. **java First** ? JVM runs the program

### **Why PATH is Required**

* Without setting PATH, you must navigate to the JDK�s `bin` folder to use `java` or `javac`.
* Adding JDK `bin` to PATH lets you run Java from **any folder**.

# **Java Program Skeleton**

---

## **1. What a Basic Java Program Looks Like**

A simple Java program (for printing �Hello World�) looks like this:

```java
import java.lang.*;

class First {
  public static void main(String[] args) {
    System.out.println("Hello World");
  }
}
```

---

## **2. Explanation of Each Line**

### **2.1 `import java.lang.*;`**

* This imports a **package**.
* A *package* in Java = collection of classes.
* `java.lang` contains essential Java classes (String, System, Math, etc).
* It is imported automatically, so this line is **optional**.

**Analogy**: like `#include <stdio.h>` in C.

---

## **3. Java Program Structure Must Have a Class**

Everything in Java must be inside a **class**.

### Example:

```java
class First {
  // contents of the class
}
```

Important rule:

### **Class name = File name**

If your class is named `First`, file must be saved as:

```
First.java
```

(Java can allow other classes in same file, but the *public* class must match filename.)

---

## **4. `main` Method**

This is the **entry point** of every Java program:

```java
public static void main(String[] args)
```

Let's break it down:

### `public`

* JVM must access this method from outside the class.
* So it must be visible ? hence `public`.

### `static`

* JVM does **not** create an object of your class.
* It directly calls the method using the class name.
* `static` allows calling without creating an object.

### `void`

* `main` does not return anything; hence return type is `void`.

### `String[] args`

* Command-line arguments.
* Even if you don't use it, **you must write it**.

---

## **5. Printing Output**

```java
System.out.println("Hello World");
```

Breakdown:

* **System** : a built-in class (starts with capital S)
* **out** : an object (standard output stream)
* **println()** : method used to print values and move to next line

Structure:

```
ClassName.objectName.methodName()
```

---

## **6. How Compilation Works**

Assume file name is `First.java`.

### **Step 1: Compile**

Run command:

```
javac First.java
```

This generates:

```
First.class
```

The `.class` file contains **bytecode**, not machine code.

---

## **7. How Execution Works**

Run command:

```
java First
```

Notes:

* Do **NOT** write `First.class`
* JVM will internally load bytecode from `.class` file.

The JVM interprets bytecode into machine code and executes it.

---

## **8. Summary: Why Each Keyword is Mandatory**

| Keyword         | Why Required                                      |
| --------------- | ------------------------------------------------- |
| `class`         | Java requires everything inside a class           |
| `public`        | JVM must access main()                            |
| `static`        | JVM calls main() without creating object          |
| `void`          | main() does not return value                      |
| `String[] args` | command-line argument container (required syntax) |

---

## **9. Big Picture of Program Flow**

1. **Write code**  `First.java`
2. **Compile**  `javac First.java` ? produces `First.class`
3. **JVM executes**  `java First`
4. JVM calls `First.main()`
5. Program output printed.

### **Compile**

```

```

? Creates `First.class` (bytecode)

### **Run**

```
java First
```

Do **not** write `.class` or `.java` while running.

## **4. File Name vs Class Name Rules**

###  Case 1: Class name = file name

```
File: First.java
class First { }
```

Everything works normally.

---

### Case 2: Class name <> file name (class NOT public)

```
File: First.java
class Second { }
```

Allowed because class is **not public**.

Compilation:

```
javac First.java
```

Creates:

* `First.java`
* `Second.class`

Run:

```
java Second
```

---

### Case 3: Class is public but filename is different

```
File: First.java
public class Second { }
```

Error:

```
class Second is public, should be declared in a file named Second.java
```

**Rule:**
If a class is **public**, the file name MUST match the class name exactly.

## **5. What Happens If You Change the main() Method?**

### Missing arguments:

```java

```

Compile & Run, you get Error:

```

public static void main(String[] args)
```

Therefore, this signature must always be written.

#  **7. Reading From Keyboard in Java Using Scanner**

## **1. Import the Scanner Class**

`Scanner` belongs to the **java.util** package.
Without importing, compiler gives **cannot find symbol: class Scanner**.

```java
import java.util.Scanner;
```

---

## **2. Creating a Scanner Object**

To read input from keyboard:

```java
Scanner sc = new Scanner(System.in);
```

* `System.in` ? built-in object already connected to keyboard
* `System.out` ? connected to monitor (used for printing)

---

## **3. Reading an Integer**

```java
int x = sc.nextInt();
System.out.println(x);
```

**Output**

* It waits for user to type a number and press **Enter**.

---

## **4. Reading a Float**

```java
float f = sc.nextFloat();
System.out.println(f);
```

---

## **5. Reading a Single Word**

```java
String s = sc.next();
```

* Reads **only one word** (stops at space).

Example input:
`Hello World` ? captures only `"Hello"`

---

## **6. Reading a Full Line**

```java
String s = sc.nextLine();
```

* Reads **entire line including spaces** until Enter is pressed.

---

## **7. Adding Two Numbers � Example Program**

```java
import java.util.*;

class ReadKeyboard {
  public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);

    int x, y;
    System.out.println("Enter two numbers:");
    x = sc.nextInt();
    y = sc.nextInt();

    int z = x + y;
    System.out.println("Sum = " + z);
  }
}
```

---

## **8. InputMismatchException**

If the input type does not match the method:

Example:

```java
int x = sc.nextInt();
```

User enters:
`2.5`

Result:

```
java.util.InputMismatchException
```

Because `nextInt()` expects only integers.

---

## **9. Viewing All Scanner Methods (Useful Trick)**

Use Java help command:

```
javap java.util.Scanner
```

This shows all available methods such as:

* `nextInt()`
* `nextFloat()`
* `nextDouble()`
* `nextLong()`
* `nextLine()`
* `next()`
* `nextByte()`
* `useRadix(int)`
* and many more�

---

## **10. Using `useRadix()` � Reading Another Number System**

You can read numbers using binary, octal, etc.

```java
sc.useRadix(2);  // binary
int x = sc.nextInt();
System.out.println(x);
```

Example Run:

Input:

```
1010
```

Output:

```
10
```

Because `10102 = 1010`

If user enters an invalid binary:

```
123
```

Error:

```
InputMismatchException
```

---

## **11. Example With useRadix()**

```java
import java.util.*;

class ReadKeyboard {
  public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);

    sc.useRadix(2);  // expect binary input

    int x = sc.nextInt();
    System.out.println("Decimal: " + x);
  }
}
```
