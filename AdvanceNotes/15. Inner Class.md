# **Java Inner Classes (Nested Classes)**

An **inner class** is a **class defined inside another class** (also called a **nested class**). Inner classes help **reduce the complexity** of large classes and allow logical grouping of classes that are only relevant to the outer class.

---

## **Why Use Inner Classes?**

* To simplify a large and complex outer class.
* To group related functionality inside one class.
* To encapsulate classes that are only used by the outer class.

---

## **Types of Inner Classes**

1. **Nested Inner Class** (non-static)
2. **Local Inner Class**
3. **Anonymous Inner Class**
4. **Static Inner Class**

---

## **Example: Nested Inner Class**

```java
class Outer {
    int x = 10;

    void outerDisplay() {
        System.out.println("Outer X: " + x);
        Inner i = new Inner();
        i.innerDisplay(); // Using inner class inside outer class
    }

    class Inner {
        int y = 20;

        void innerDisplay() {
            System.out.println("Outer X: " + x); // Can access outer class members directly
            System.out.println("Inner Y: " + y);
        }
    }
}

public class Test {
    public static void main(String[] args) {
        Outer o = new Outer();
        o.outerDisplay(); // Outer class uses inner class internally
    }
}
```

---

## **Key Points**

1. **Inner class → Outer class members**: Can **directly access** all members of outer class, even private ones.
2. **Outer class → Inner class members**: Cannot access **directly**. Must **create an inner class object**:

   ```java
   Inner i = new Inner(); // inside outer class
   System.out.println(i.y);
   ```
3. **Outside the outer class**:

   * You **cannot directly create an inner class object** without an outer class object.

   ```java
   Outer.Inner i = new Outer().new Inner();
   ```
4. **Compilation**:

   * Outer class → `Outer.class`
   * Inner class → `Outer$Inner.class`

---

✅ **Summary of Access Rules**

| Relationship              | Access to Outer Members | Access to Inner Members |
| ------------------------- | ----------------------- | ----------------------- |
| Inner class → Outer class | Direct access           | -                       |
| Outer class → Inner class | -                       | Through inner object    |
| Other class → Inner class | -                       | Through outer object    |

---


## **1. Outer and Inner Class Example**

You have:

```java
class Outer {
    int x = 10; // Outer class member

    Inner i = new Inner(); // Inner class object as a member of Outer

    void outerDisplay() {
        i.innerDisplay(); // Using inner class method
    }

    class Inner {
        int y = 20; // Inner class member

        void innerDisplay() {
            System.out.println("Outer X: " + x); // Can access outer class member directly
            System.out.println("Inner Y: " + y); // Access its own member
        }
    }
}

public class NestedInner {
    public static void main(String[] args) {
        Outer o = new Outer();
        o.outerDisplay(); // Outer class uses inner class internally

        // Directly creating Inner object outside Outer class
        Outer.Inner oi = o.new Inner();
        oi.innerDisplay(); // Accessing Inner class method from main
    }
}
```

---

## **2. Key Observations from Demonstration**

1. **Creating Inner Objects Inside Outer**

   * You can create an object of an inner class **inside an outer class method**:

     ```java
     Inner i = new Inner();
     i.innerDisplay();
     ```
   * Or as a **member of the outer class**, making it accessible in all methods of the outer class.

2. **Accessing Outer Members**

   * Inner class can **directly access outer class members**, including private members.
   * Example: `System.out.println(x);` inside `Inner`.

3. **Accessing Inner Members from Outer Class**

   * Outer class **cannot access inner class members directly**; it must use the inner class object:

     ```java
     System.out.println(i.y); // Accessing Inner member
     ```

4. **Accessing Inner Class Outside Outer Class**

   * Use this syntax:

     ```java
     Outer.Inner obj = outerObject.new Inner();
     ```
   * You **must first create an outer class object** to create an inner class object.

5. **Static Members of Outer Class**

   * Inner class can **directly access static members** of outer class using the class name.

---

## **3. Class Files Generated**

* **Outer class → `Outer.class`**
* **Inner class → `Outer$Inner.class`**
* **Main application class → `NestedInner.class`**

> Each class in Java generates a separate `.class` file, even inner classes.
> The `$` symbol indicates an inner class.

---

## **4. Execution Flow Recap**

1. `main` creates an object of `Outer`.
2. Outer object calls `outerDisplay()`.
3. `outerDisplay()` creates or uses inner class object and calls `innerDisplay()`.
4. Inner class prints both outer and inner members.
5. From `main`, you can also create `Outer.Inner` object directly using the outer object and call its methods.

---

✅ **Summary**

* **Inner class** simplifies code and logically groups functionality.
* Inner classes **have access to outer class members**.
* Outer class **needs an inner object** to access inner members.
* **Outer$Inner.class** file is automatically generated by the compiler.

## **1. Local Inner Class**

### **Definition**

* A **class defined inside a method** of another class.
* It is **visible only within that method**.
* Useful when a class is needed **only temporarily** inside a method.

### **Example**

```java
class Outer {
    void display() {
        class Inner { // Local inner class
            void innerDisplay() {
                System.out.println("Hello");
            }
        }

        Inner i = new Inner(); // Create object of local inner class
        i.innerDisplay();      // Call its method
    }
}

public class LocalInnerDemo {
    public static void main(String[] args) {
        Outer o = new Outer();
        o.display(); // Prints "Hello"
    }
}
```

**Key Points**

* Cannot be accessed outside the method.
* Useful for **temporary tasks**, like implementing an interface or extending a class **inside a method**.

---

## **2. Anonymous Inner Class**

### **Definition**

* A **class without a name**.
* Defined **at the point of object creation**.
* Commonly used to:

  * Implement **interfaces**.
  * Extend **abstract classes**.
* Allows creating an object **and overriding methods** at the same time.

### **Example with Abstract Class**

```java
abstract class My {
    abstract void display();
}

public class AnonymousDemo {
    public static void main(String[] args) {
        My m = new My() {  // Anonymous inner class
            void display() {
                System.out.println("Hello from anonymous class");
            }
        };
        m.display(); // Prints message
    }
}
```

### **Example with Interface**

```java
interface MyInterface {
    void display();
}

public class AnonymousInterfaceDemo {
    public static void main(String[] args) {
        MyInterface mi = new MyInterface() { // Anonymous inner class
            public void display() {
                System.out.println("Hello from anonymous interface class");
            }
        };
        mi.display(); // Prints message
    }
}
```

**Key Points**

* No separate class file name required.
* Extremely useful for **short-term usage**, especially with built-in interfaces or event handlers in Java GUI.

---

## **3. Static Inner Class** (Introduction)

* A **static nested class** is defined using the `static` keyword inside an outer class.
* Unlike normal inner classes:

  * It **cannot access non-static members** of outer class directly.
  * Can be instantiated **without an outer class object**.

> You mentioned that static inner class will be explained next—this is the setup for it.

---

✅ **Summary of Inner Class Types**

| Type                  | Where Defined                    | Access Scope                          | Can Access Outer Members |
| --------------------- | -------------------------------- | ------------------------------------- | ------------------------ |
| Nested Inner Class    | Inside outer class               | Anywhere in outer class or via object | Yes                      |
| Local Inner Class     | Inside a method of outer class   | Only inside that method               | Yes                      |
| Anonymous Inner Class | At point of object creation      | Only via its reference                | Yes                      |
| Static Inner Class    | Inside outer class with `static` | Anywhere via outer class reference    | Only static members      |

---


## **Static Inner Class**

### **Definition**

* A **static inner class** is a class defined inside an outer class with the keyword `static`.
* Unlike normal (nested) inner classes:

  * **It does not require an object of the outer class** to be instantiated.
  * Can be accessed **directly using the outer class name**.
* Static inner classes **can only access static members** of the outer class directly.
* They **cannot access non-static members** of the outer class without an outer class object.

---

### **Example**

```java
class Outer {
    static int X = 10;    // static member
    int Y = 20;           // non-static member

    static class Inner {  // static inner class
        void display() {
            System.out.println(X); // Allowed: static member
            // System.out.println(Y); // Not allowed: non-static member
        }
    }
}

public class Test {
    public static void main(String[] args) {
        Outer.Inner obj = new Outer.Inner(); // Create object directly
        obj.display();                        // Prints 10
    }
}
```

---

### **Key Points**

1. **Instantiation**

   * Nested inner class:

     ```java
     Outer.Inner i = new Outer().new Inner();
     ```

     Requires an outer class object first.
   * Static inner class:

     ```java
     Outer.Inner i = new Outer.Inner();
     ```

     Directly accessible via outer class name; only **one `new` keyword** is needed.
2. **Access**

   * Can access **static members** of outer class.
   * Cannot access **non-static members** directly.
3. **Use Case**

   * Useful when inner class functionality is related to outer class but **does not depend on an instance** of the outer class.
   * Often used in **utility or helper classes** within an outer class.

---

✅ **Comparison: Nested vs Static Inner Class**

| Feature                      | Nested Inner Class                         | Static Inner Class                   |
| ---------------------------- | ------------------------------------------ | ------------------------------------ |
| Requires outer class object? | Yes                                        | No                                   |
| Access outer class members   | All members                                | Only static members                  |
| Instantiation syntax         | `Outer.Inner i = new Outer().new Inner();` | `Outer.Inner i = new Outer.Inner();` |
| Use case                     | Dependent on outer instance                | Independent helper/utility           |

## **1. Local Inner Class**

* **Definition**: A class defined **inside a method** of an outer class.
* **Scope**: Only visible and usable **within that method**.
* **Example**:

```java
class Outer {
    public void display() {
        class Inner {
            void show() {
                System.out.println("Hello");
            }
        }
        Inner i = new Inner();
        i.show();
    }
}

public class Test {
    public static void main(String[] args) {
        Outer o = new Outer();
        o.display();  // Output: Hello
    }
}
```

* **Anonymous Object**: Can call method without storing a reference:

```java
new Inner().show();  // Output: Hello
```

* **Use case**: When a class is needed **only within a method**, e.g., implementing an interface locally.

---

## **2. Anonymous Inner Class**

* **Definition**: A **class without a name** created at the time of instantiation.
* Can **inherit an abstract class** or **implement an interface** in a single line.
* **Example (abstract class)**:

```java
abstract class My {
    abstract void show();
}

Outer o = new Outer();
My m = new My() {
    void show() {
        System.out.println("Hello");
    }
};
m.show(); // Output: Hello
```

* **Example (interface)**:

```java
interface My {
    void show();
}

My m = new My() {
    public void show() {
        System.out.println("Hello");
    }
};
m.show(); // Output: Hello
```

* **Use case**: Quickly implement an interface or abstract class when the class is **used only once**.

---

## **3. Static Inner Class**

* **Definition**: A class inside an outer class declared with `static`.

* **Key points**:

  * Does **not require an outer class object** to instantiate.
  * Can access **only static members** of the outer class.
  * Can be instantiated **directly using outer class name**.

* **Example**:

```java
class Outer {
    static int x = 20;
    int y = 10;

    static class My {
        void show() {
            System.out.println(x); // Allowed
            // System.out.println(y); // Not allowed
        }
    }
}

public class Test {
    public static void main(String[] args) {
        Outer.My m = new Outer.My();
        m.show();  // Output: 20
    }
}
```

* **Difference from nested inner class**:

  * Nested inner class **needs outer class object**.
  * Static inner class **does not** need outer class object.

---

### ✅ **Summary Table: Inner Classes**

| Inner Class Type      | Scope/Access                 | Object Creation                            | Example Use Case                              |
| --------------------- | ---------------------------- | ------------------------------------------ | --------------------------------------------- |
| Local Inner Class     | Inside a method only         | `Inner i = new Inner();`                   | Helper class used only in method              |
| Anonymous Inner Class | No name, used once           | `new My() { ... };`                        | Implement interface or abstract class quickly |
| Nested Inner Class    | Member of outer class        | `Outer.Inner i = new Outer().new Inner();` | Depends on outer instance                     |
| Static Inner Class    | Static member of outer class | `Outer.My m = new Outer.My();`             | Independent helper class                      |
