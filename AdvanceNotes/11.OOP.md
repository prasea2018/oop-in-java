# **ðŸ“˜ DATA HIDING in Object-Oriented Programming (OOP)**

## **1. Why Do We Need Data Hiding?**

Data hiding exists to **protect internal data** of an object from being misused, changed incorrectly, or accessed directly from outside the class.

In real-world engineered products, users cannot directly operate internal componentsâ€”they only interact through simple external interfaces (buttons, controls).
This principle is applied in software as well.

---

## **2. Real-Life Examples of Data Hiding**

### **Example 1: Television**

* You can see:

  * Power button
  * Volume control
  * Channel buttons
* You **cannot see** the internal circuitry.
* Pressing a button triggers an operation *inside* the circuit without exposing internal components.

**Interpretation:**

* **Circuit** â†’ internal data
* **Buttons** â†’ methods
* Internal mechanism â†’ hidden data
* Allowed operations â†’ visible methods

---

### **Example 2: Car Engine**

* Driver uses **steering**, **accelerator**, **brake**.
* Actual work happens inside the **engine**.
* Engine is covered under a **bonnet**, so users cannot directly touch internal parts.

**Interpretation:**

* **Engine** â†’ data
* **Pedals & steering** â†’ methods
* Engine hidden â†’ data hiding
* Operations available â†’ public methods

---

## **3. How Does Data Hiding Work in Software?**

When designing software classes:

### **Data = properties/variables**

### **Methods = operations/functions**

Example:
In a `Rectangle` class

* Properties: `length`, `breadth`
* Methods: `area()`, `perimeter()`

Methods operate *on* data â€” same as buttons operating on circuitry.

---

## **4. How to Hide Data in Java**

To hide data, we use:

### **`private` access modifier**

```java
class Rectangle {
    private int length;
    private int breadth;

    public int area() { ... }
    public int perimeter() { ... }
}
```

### What does `private` do?

* Properties cannot be accessed directly outside the class.
* They can only be accessed **inside** the class.

---

## **5. Problem Without Data Hiding**

If `length` and `breadth` are public:

```java
Rectangle r = new Rectangle();
r.length = -10;   // invalid
r.breadth = 5;
```

This leads to incorrect values causing invalid area/perimeter.

Thus, data must be protected.

---

## **6. How to Access Hidden Data? (Getters and Setters)**

To read and modify private data, we provide **property methods**:

## **Getters (read)**

Return the value.

```java
public int getLength() {
    return length;
}
```

## **Setters (write)**

Set the value â€” can include validation.

```java
public void setLength(int length) {
    if (length > 0)
        this.length = length;
    else
        this.length = 0; // validation
}
```

### Why validation in setter?

* Prevent invalid values (e.g., negative length)
* Protect data integrity

---

## **7. Using Getters and Setters**

Example:

```java

r.setLength(10);
r.setBreadth(5);

System.out.println(r.area());
System.out.println(r.perimeter());
```

If user writes:

```java

```

Setter converts it to:

```java

```

Because negative length is invalid.

This is how **controlled access** works.

---

## **8. Summary of Data Hiding**

âœ” Data hiding protects internal object data.
âœ” Achieved using `private` variables.
âœ” Access provided via **public getters and setters**.
âœ” Prevents misuse, enforces validation, improves security.
âœ” Makes classes more organized and maintainable.

A **read-only property** means:

When a property must **never change** after object creation.

A **write-only property** means:

âœ” Only setter is allowed
âœ– No getter method

### When is it needed?

This is rare in real life, but useful in large software when:

* One class writes data
* Another class reads data
* Known as **producerâ€“consumer pattern**

## **2ï¸âƒ£ What is a Constructor?**

A **constructor is a special method of a class** that is automatically called **whenever an object is created**.

Example:

```java

```

Here, `new Rectangle()` automatically calls the constructor of the `Rectangle` class.

## **3ï¸âƒ£ Characteristics of a Constructor**

### âœ” Must have the **same name as the class**

```java

    public Rectangle() { }

```

### âœ” Has **no return type**, not even void

```java

public void Rectangle() {}   // âŒ NOT a constructor
```

### âœ” Usually declared as **public**

(But private constructors are also possible.)

## **4ï¸âƒ£ Default Constructor**

If you do NOT create any constructor, Java provides one automatically.

### **A. Non-Parameterized Constructor**

A constructor that takes **no parameters**.

### **B. Parameterized Constructor**

A constructor that takes **arguments**.

Usage:


This sets:

```

breadth = 5
```

## **6ï¸âƒ£ Constructor Overloading**

You can write **multiple constructors** in the same class, as long as their parameter lists differ.

Example:

```java

        length = 1;
        breadth = 1;
    }

    public Rectangle(int l, int b) {   // Two parameters
        length = l;
        breadth = b;
    }
}
```

This is perfectly valid because of **method overloading** rules.

âœ” Same name
âœ” Different parameters
