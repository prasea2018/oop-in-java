## **1. Definition of Java**

**Java** is a **high-level, object-oriented, platform-independent programming language** developed by **Sun Microsystems** (now owned by **Oracle Corporation**).

## **Key Features(Buzzword) of Java**

* **Object-Oriented** â€“ Uses classes and objects
* **Platform Independent** â€“ *Write Once, Run Anywhere (WORA)*
* **Secure** â€“ No direct memory access
* **Robust** â€“ Strong memory management and exception handling
* **Multithreaded** â€“ Supports concurrent execution
* **Portable** â€“ Same bytecode runs on any JVM


#### **1. Simple**

* Java syntax is similar to **C** and **C++**, making it easy for programmers coming from those languages.
* Even beginners can learn Java easily because the syntax is clean and English-like.
* Unnecessary complex features of C/C++ (like pointers, operator overloading, multiple inheritance of classes) are removed to keep Java simple.

#### **2. Secure**

* Java provides strong security because programs run inside the **Java Virtual Machine (JVM)**.
* JVM uses:

  * **Class Loader**
  * **Bytecode Verifier**
  * **Security Manager**
* These components ensure:

  * No malicious code is executed
  * No unauthorized memory access
  * Programs cannot cause virus-like behavior

Thus, Java programs run safely and are protected from malware.


#### **3. Object-Oriented**

* Java supports all OOP principles:

  * **Abstraction**
  * **Encapsulation**
  * **Inheritance**
  * **Polymorphism**
* OOP makes programs modular, reusable, flexible, and easy to maintain.
* Objects and classes help model real-world entities.

#### **3. Platform Independent**

* Java is independent of hardware architecture.
*Write Once, Run Anywhere (WORA)*
* It does not depend on:

  * Von Neumann architecture
  * ARM architecture
  * RISC / CISC CPUs
* JVM acts as a layer between Java code and the underlying processor.
* Therefore, Java runs on any architecture where JVM is available.

#### **4. Robust**

* Java is strong and reliable.
* It handles run-time errors using **exception handling**.
* It removes memory errors by:

  * Automatic **Garbage Collection**
  * No pointer manipulation
* Even if resources are unavailable at run-time, Java programs do not crash easily.


#### **5. Multithreaded**

* Java supports **multithreading**, meaning:

  * A program can be broken into multiple threads.
  * All threads run **simultaneously**.
* Helpful for:

  * Animations
  * Games
  * Servers
  * Concurrent processing



## **6. Interpreted**

* Java uses an **interpreter** (JVM) to execute bytecode line-by-line.
* Earlier Java versions were slow due to interpretation only.
* Later, **JIT compiler (Just-In-Time)** was added to improve performance.

## **4. Applications and Applets**

### **Java Applications**

* Standalone programs
* Run on a local machine
* Have a `main()` method


```java

  public static void main(String[] args) {
    System.out.println("Hello Java Application");
  }
}
```

---

### **Java Applets**

* Small Java programs
* Embedded in web pages
* Run inside a browser (obsolete now)

**Difference Between Application and Applet**

| Application        | Applet              |
| ------------------ | ------------------- |
| Runs independently | Runs inside browser |
| Uses `main()`      | No `main()` method  |
| More powerful      | Restricted access   |

**Note:** Applets are no longer supported in modern browsers, but included for **academic understanding**.

## **5. Java Virtual Machine (JVM)**

### **What is JVM?**

**JVM (Java Virtual Machine)** is an abstract machine that:

* Executes Java **bytecode**
* Converts bytecode into machine code
* Makes Java platform-independent

**Key Role:**

> JVM acts as a **translator between Java bytecode and the operating system.**

---

### **JVM Architecture**

* Class Loader
* Memory Area
* Execution Engine
* Garbage Collector

## **6. Bytecode â€“ Not an Executable Code**

### **What is Bytecode?**

* Intermediate code generated after compilation
* Stored in `.class` file
* Cannot run directly on hardware

**Important Point:**

> **Bytecode is NOT machine code. It requires JVM to execute.**

## **7. Procedure-Oriented vs Object-Oriented Programming**

### **Procedure-Oriented Programming (POP)**

* Focus on functions
* Data is exposed
* Example: C

### **Object-Oriented Programming (OOP)**

* Focus on objects
* Data hiding using encapsulation
* Example: Java

**Comparison Table**

| POP                   | OOP              |
| --------------------- | ---------------- |
| Function-based        | Object-based     |
| Less secure           | More secure      |
| Difficult to maintain | Easy maintenance |

## **8. Compiling and Running a Simple Java Program**

### **Steps to Compile and Run**

1. Write program in a `.java` file
2. Compile using:

3. Run using:
